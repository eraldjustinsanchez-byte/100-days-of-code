# 100 Days of Code – Backend Engineering Journey

## Overview

This repository documents my **100-day backend engineering journey**.

The goal is not just to "complete projects", but to:

- Build real backend systems
- Master data persistence (Files → SQLite → SQL Databases)
- Understand automation & scripting (Bash)
- Apply structured repetition (spaced rebuild system)
- Develop long-term retention, not short-term memory

This challenge focuses on **consistency, system design, and deep understanding**.

> Discipline over motivation.  
> Structure over randomness.  
> Execution over perfection.

---

## Objectives

By the end of 100 days, I aim to:

- Think like a backend engineer
- Design relational databases confidently
- Build CLI-based backend systems
- Understand CRUD architecture deeply
- Work with structured data (CSV, JSON, SQL)
- Automate workflows using Bash scripting
- Apply spaced repetition to retain everything long-term

---

## Learning Method

I follow a structured system:

### New Project Every Day
Each day introduces a new backend concept or system.

### Spaced Rebuild System
Every project is rebuilt:
- +3 days
- +6 days
- +12 days
- +24 days

Rebuilding ensures:
- Deep retention
- Structural understanding
- Faster implementation over time

### Focus on Structure
Instead of memorizing syntax, I focus on:
- Data flow
- Logic design
- Schema structure
- Input → Processing → Output pipelines
